# Claude Code Remote Control 完全ガイド

## 概要

**Remote Control** は 2026年2月24日に発表された Claude Code の新機能で、ローカルマシンで動作中の Claude Code セッションを、スマートフォン・タブレット・別のPC のブラウザからリモート操作できる。

デスクで始めたコーディング作業を、ソファからスマホで続けたり、別の PC のブラウザから操作したりできる。**コードやファイルがクラウドに転送されることはなく、すべてローカルマシン上で実行される**点が最大の特徴。

## 「Claude Code on the web」との違い

|  | Remote Control | Claude Code on the web |
|---|---|---|
| **実行環境** | ローカルマシン | Anthropic のクラウド |
| **ファイルシステム** | ローカルのファイルに直接アクセス | クラウド上のサンドボックス |
| **MCP サーバー** | ローカルの MCP サーバーが使える | 使えない |
| **Skills / プロジェクト設定** | すべて利用可能 | 利用不可 |
| **リポジトリ** | ローカルにクローン済みのもの | クローン不要（クラウドで取得） |
| **用途** | 作業中のローカル環境を別デバイスから継続 | ローカル環境なしで新規タスクを開始 |

## 対応プラン

- **Max プラン**: 利用可能（リサーチプレビュー）
- **Pro プラン**: 近日対応予定
- **Team / Enterprise プラン**: 現時点では未対応
- **API キー**: 未対応

## 前提条件

1. Claude Code がインストール済みであること
2. `claude` コマンドで `/login` によるclaude.ai 認証が済んでいること
3. プロジェクトディレクトリで最低1回 `claude` を実行し、ワークスペース信頼ダイアログを承認済みであること

## 起動方法

### 方法1: 新規セッションとして起動（`claude remote-control` / `claude rc`）

```bash
cd /path/to/project
claude remote-control
```

ターミナルにセッション URL と QR コードが表示される。スペースキーで QR コードの表示/非表示を切り替え可能。

**対応フラグ:**

| フラグ | 説明 |
|---|---|
| `--verbose` | 詳細な接続・セッションログを表示 |
| `--sandbox` | サンドボックスモードを有効化（ファイルシステム・ネットワーク分離） |
| `--no-sandbox` | サンドボックスを無効化（デフォルト） |

### 方法2: 既存セッションから起動（`/remote-control` / `/rc`）

既に Claude Code セッション内で作業中の場合、スラッシュコマンドで切り替える:

```
/remote-control
```

または短縮形:

```
/rc
```

**重要な違い:**

| 観点 | `claude rc`（ターミナルコマンド） | `/rc`（スラッシュコマンド） |
|---|---|---|
| コンテキスト | 新規セッションを開始 | 既存セッションの会話履歴を引き継ぐ |
| フラグ | `--sandbox` 等を指定可能 | フラグ指定不可 |
| 用途 | 新しいリモートタスクの開始 | 進行中の作業をモバイルに持ち出す |

**Tips:** `/rc` を実行する前に `/rename` でセッションに分かりやすい名前をつけておくと、複数デバイスからセッションを探しやすくなる。

```
/rename feature-auth-refactor
/rc
```

## 接続方法（クライアント側）

Remote Control セッションが起動したら、以下のいずれかの方法で接続する:

1. **セッション URL を開く** — ターミナルに表示された URL をブラウザで開く → [claude.ai/code](https://claude.ai/code) でセッションに直接接続
2. **QR コードをスキャン** — Claude モバイルアプリで QR コードをスキャンして接続
3. **claude.ai/code またはモバイルアプリのセッション一覧** — Remote Control セッションは PC アイコン + 緑のステータスドットで表示される

Claude モバイルアプリ未インストールの場合は、Claude Code 内で `/mobile` コマンドを実行するとダウンロード用 QR コードが表示される。

## 全セッションで自動有効化

デフォルトでは手動起動が必要だが、すべてのセッションで自動的に Remote Control を有効にすることもできる:

1. Claude Code 内で `/config` を実行
2. 「Enable Remote Control for all sessions」を `true` に設定

## セキュリティモデル

Remote Control は「ローカルマシンへの外部からの侵入経路を作らない」ことを設計原則としている。通常の Claude Code セッションと同じセキュリティ境界を維持しつつ、リモートアクセスを実現する仕組みになっている。

### 通信アーキテクチャ

```
[スマホ/ブラウザ] ──HTTPS──→ [Anthropic API] ←──HTTPS(ポーリング)── [ローカル Claude Code]
                              (メッセージ中継)
```

Remote Control の通信は **3者間のリレー方式**で動作する:

1. **ローカル Claude Code プロセス**が Anthropic API に対してアウトバウンド HTTPS 接続を張り、ワークをポーリングする
2. **スマホ/ブラウザ**が Anthropic API にプロンプトを送信する
3. **Anthropic API** がメッセージを中継（ストリーミング接続経由）し、双方を橋渡しする

ローカルマシン上にリスニングポート（インバウンドポート）は一切開かれない。すべての通信はローカル側からの**アウトバウンド HTTPS のみ**で成立する。

### トランスポートセキュリティ

すべてのトラフィックは **TLS（Transport Layer Security）** で暗号化される。これは通常の Claude Code セッションが Anthropic API と通信する際と同じトランスポートセキュリティであり、Remote Control 固有の追加プロトコルは使用しない。

### クレデンシャル管理

Remote Control の接続には**複数の短寿命クレデンシャル**が使用される:

- 各クレデンシャルは**単一の目的にスコープ限定**されている（認証用、セッション識別用、メッセージルーティング用など）
- それぞれが**独立して失効**する
- 1つのクレデンシャルが漏洩した場合の**影響範囲（blast radius）を最小化**する設計

ローカル側の認証情報（API キーやトークン）は macOS では **macOS Keychain** に暗号化して保存される。

### パーミッションモデル

Remote Control 経由の操作にも、通常の Claude Code と同じパーミッションシステムが適用される:

- **デフォルトは読み取り専用** — ファイル編集やコマンド実行には明示的な許可が必要
- **書き込み制限** — Claude Code を起動したフォルダとそのサブフォルダにのみ書き込み可能。親ディレクトリへの書き込みは許可なしではできない
- **コマンド承認** — bash コマンドの実行前に承認が求められる
- **ネットワークリクエスト承認** — ネットワークリクエストを行うツールはデフォルトでユーザー承認が必要

ただし、Simon Willison の検証によると、`--dangerously-skip-permissions` フラグは Remote Control では認識されないため、すべてのアクションに対して手動承認が必要になるとのこと。

### ワークスペース信頼

Remote Control を使用するには、事前にローカルでそのプロジェクトディレクトリの**ワークスペース信頼ダイアログを承認**している必要がある。未承認のディレクトリに対してリモートからアクセスすることはできない。

### データフローとプライバシー

Remote Control セッションのデータフローは**ローカル Claude Code と同じ扱い**になる（クラウド実行とは異なる）:

| 項目 | Remote Control | Claude Code on the web |
|---|---|---|
| **コード実行** | ローカルマシン上 | クラウド VM 上 |
| **VM / サンドボックス** | なし（ローカル実行） | Anthropic 管理の隔離 VM |
| **API 経由で流れるデータ** | プロンプトとモデル出力（通常の Claude Code と同じ） | 同左 + リポジトリクローン |
| **監査ログ** | なし（ローカル） | クラウド環境では自動記録 |

### データ保持ポリシー

Remote Control セッションのデータ保持は、利用しているプランに応じた通常のポリシーが適用される:

| プラン | モデル改善への利用を許可 | 保持期間 |
|---|---|---|
| Max（許可あり） | はい | 5年間 |
| Max（許可なし） | いいえ | 30日間 |

プライバシー設定は [claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls) からいつでも変更可能。

### テレメトリ

Remote Control セッション中も以下のテレメトリが送信される（通常の Claude Code と同様）:

| サービス | 内容 | オプトアウト |
|---|---|---|
| **Statsig** | レイテンシ・信頼性・使用パターン（コードやファイルパスは含まない） | `DISABLE_TELEMETRY=1` |
| **Sentry** | エラーログ | `DISABLE_ERROR_REPORTING=1` |

いずれも TLS 暗号化（転送時）および AES-256 暗号化（保存時）が適用される。

### セキュリティ上の注意点

- Remote Control は **Anthropic API を経由するリレー方式**であり、P2P 接続ではない。Anthropic のインフラを信頼することが前提
- セッション URL や QR コードを第三者に共有すると、そのセッションにアクセスされる可能性がある。**URL の取り扱いには注意**が必要
- ローカルマシンの物理セキュリティは利用者の責任。ターミナルが開いたまま放置されれば、Remote Control の有無にかかわらずリスクがある
- MCP サーバーのセキュリティは Anthropic の管理外。Remote Control 経由でも MCP サーバーが利用可能なため、信頼できる MCP サーバーのみを設定すること

## 制約事項

| 制約 | 詳細 |
|---|---|
| **同時接続数** | 1つの Claude Code セッションにつきリモート接続は1つまで |
| **ターミナルを閉じると終了** | ローカルプロセスとして動作するため、ターミナルを閉じたり `claude` プロセスを停止するとセッション終了 |
| **ネットワーク切断** | スリープからの復帰は自動再接続されるが、約10分以上ネットワーク不通が続くとタイムアウトして終了 |
| **OS シャットダウン** | マシンの電源を切るとセッションは永続的に終了（スリープとは異なる） |
| **リモートからの起動不可** | まずローカルでセッションを開始する必要がある。リモートから新規セッションを立ち上げることはできない |

## 実用的なユースケース

### 1. 長時間タスクのモニタリング
大規模なリファクタリングやテスト実行をデスクで開始し、離席後もスマホから進捗を確認・指示を送る。

### 2. コードレビューの継続
デスクで PR のレビューを始め、移動中にスマホからコメントの追加やレビューの完了を行う。

### 3. デバッグの続行
問題調査をデスクで開始し、ログやスタックトレースの確認をタブレットから続ける。

### 4. ペアプログラミング的な使い方
ターミナルとモバイルの両方からメッセージを送れるため、会話が同期された状態でマルチデバイスで作業できる。

## コマンドリファレンス

```bash
# 新規 Remote Control セッションを開始
claude remote-control

# 短縮形
claude rc

# 詳細ログ付きで開始
claude remote-control --verbose

# サンドボックスモードで開始
claude remote-control --sandbox

# 既存セッション内で Remote Control を開始（スラッシュコマンド）
/remote-control
/rc

# セッション名を設定（/rc の前に実行推奨）
/rename my-session-name

# Claude モバイルアプリのダウンロード QR を表示
/mobile
```
